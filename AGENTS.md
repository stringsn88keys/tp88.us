# AGENTS.md — AI Agent Guidance for tp88.us

This file describes conventions for AI coding agents (Claude Code, Copilot, etc.) working on this repository.

---

## Site Overview

`tp88.us` is a static personal site served from S3/CloudFront. All HTML is either:

1. **Hand-authored** — written directly and committed as-is.
2. **Generated by `index_me.rb`** — produced at publish time from source files.

Run `ruby index_me.rb` to regenerate all generated files and sync to S3.

## SEO: Script Metadata

`index_me.rb` extracts descriptions from script source files to use in the index listing and as `<meta name="description">` in the generated HTML page.

### Ruby (`.rb`) and Shell (`.sh`) scripts

Add a `##` comment on a single line near the top of the file:

```ruby
## Short one-line description of what this script does
```

```bash
## Short one-line description of what this script does
```

The first `##` comment line is used as the description. Place it before any `require` statements or code.

### PowerShell (`.ps1`) scripts

Use PowerShell comment-based help. `index_me.rb` looks for `.SYNOPSIS` first:

```powershell
<#
.SYNOPSIS
  Short one-line description of what this script does.
#>
```

If no `.SYNOPSIS` block is present, it falls back to the first `##` comment line.

### Batch files (`.bat`)

Not currently extracted. Add `## description` as a `REM ## description` comment if desired and update `file_description` in `index_me.rb`.

## SEO: Blog Posts (Markdown)

Blog posts in `blog/*.md` are rendered to HTML by `index_me.rb`. SEO metadata is extracted automatically:

- **Title**: first `# Heading` in the markdown.
- **Description**: first non-heading paragraph text (truncated to 160 chars).
- **Keywords**: title + filename words.

To improve the auto-generated description, start your markdown with a clear, informative paragraph right after the `# Title` heading.

## robots.txt and sitemap.xml

Both are **generated by `index_me.rb`** and pushed to S3 on each run. Do **not** commit hand-authored versions — they will be overwritten.

To add a new page to the sitemap, add it to the `sitemap_urls` array in `index_me.rb` (or ensure it matches one of the `Dir[]` globs already there).

To block a path from crawlers, edit the `robots.txt` heredoc in `index_me.rb`.

## Publishing

```bash
ruby index_me.rb
# Optional: force CloudFront cache invalidation
DO_INVALIDATION=1 ruby index_me.rb
```

`index_me.rb` will:
1. Render `.erb` calculator files to `.html`
2. Generate syntax-highlighted HTML for all scripts in `scripts/`
3. Render `blog/*.md` to HTML
4. Refresh `data/coffee.csv` from Google Sheets
5. Generate `books_all.html`, `books_completed.html`, `coffee.html`
6. Add Google Analytics to generated HTML files
7. Generate `index.html`
8. Write `robots.txt` and `sitemap.xml`
9. Sync everything to `s3://tp88.us`
10. (Optional) Invalidate the CloudFront distribution

## Adding New Content

### New script
1. Add the script file to `scripts/`.
2. Add a `##` description comment (or `.SYNOPSIS` for `.ps1`) at the top.
3. Run `index_me.rb` — the highlighted HTML page and index entry are created automatically.

### New blog post
1. Create `blog/my-post.md` with a `# Title` heading.
2. Write an informative opening paragraph (used as the SEO description).
3. Run `index_me.rb`.

### New standalone HTML calculator or page
1. Create the HTML file with full SEO meta tags (see template above).
2. If it should appear in `index.html`, add a `Dir[]` glob or a manual link in `index_me.rb`.
3. If it should appear in the sitemap, add it to the `sitemap_urls` array in `index_me.rb`.
